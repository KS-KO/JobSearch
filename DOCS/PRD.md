# Product Requirements Document (PRD)

## 1. 프로젝트 개요

- 프로젝트명: 연령층 기반 채용 기업 추천 시스템
- 목표: Saramin과 JobKorea에서 제공하는 검색 조건 및 연령층 필터를 기준으로, 사용자가 선호하는 조건에 맞는 기업 정보를 수집하고 정리하여 보기 쉽게 제공
- 대상 사용자: 채용 준비자, 인사 담당자, 취업 컨설턴트
- 기본 기술 방향: C# 기반 메인 서비스와 Python 기반 수집기를 조합한 하이브리드 구조를 채택
- 프레임워크 및 런타임 정책: 구현 시점 기준 최신의 안정적인 버전을 우선 적용
- UI 구현 방향: C# 데스크톱 클라이언트는 WPF 기반으로 구현

## 2. 배경 및 문제 정의

- Saramin과 JobKorea는 국내 주요 채용 플랫폼이지만, 여러 검색 조건을 동시에 비교하기 어렵고 연령층별 적합 기업을 빠르게 찾기 어려움
- 사용자들은 특정 연령대(예: 20대, 30대, 40대, 50대 이상)에 맞는 기업 정보와 채용 공고를 효율적으로 탐색하고 싶어함
- 플랫폼별 검색 조건을 통합 정리하여 기업 목록을 제공하면 시간 절약과 비교 분석이 용이함

## 3. 주요 기능

### 3.1 연령층 필터링
- Saramin과 JobKorea에서 제공하는 연령층 지정 조건을 파악
- 대상 연령대별(예: 20대, 30대, 40대, 50대 이상)로 기업을 분류
- 연령층별 추천 기업 리스트 제공

### 3.2 검색 조건 기반 기업 정리
- 주요 검색 조건:
  - 산업군/직무
  - 경력(신입, 경력)
  - 근무 형태(정규직, 계약직, 인턴)
  - 지역
  - 연봉/급여 수준
  - 기업 규모
- 각 플랫폼별 검색 결과를 정리하여 비교 가능한 형태로 제공

### 3.3 기업 정보 구성
- 기업명
- 채용 공고 제목
- 연령층 적용 여부 및 적합도
- 주요 채용 조건
- 거래 채용 플랫폼(Saramin 또는 JobKorea)
- 기업 위치 및 채용 형태
- 추가 메모 및 특징 요약

### 3.4 문서화 및 공유
- DOCS 폴더 내에 정리된 보고서 형태로 저장
- 플랫폼별, 연령층별, 조건별 정리 문서 제공
- 향후 확장: 웹/앱 UI, API 연동, 자동 업데이트 기능

### 3.5 시스템 구성 방향
- 메인 서비스는 C#(.NET) 기반으로 구성하여 기업 정보 조회, 정리, API 제공, 문서화 결과 관리 기능을 담당
- Python 수집기는 Saramin과 JobKorea의 검색 결과 수집, 전처리, 정규화 작업을 담당
- 수집기와 메인 서비스는 파일(JSON/CSV) 또는 API 기반 인터페이스로 연동 가능하도록 설계
- 향후 수집기 확장 시에도 메인 서비스의 비즈니스 로직을 변경하지 않도록 역할을 분리
- C#과 Python 양쪽 모두 프레임워크 선택 시 LTS 또는 이에 준하는 최신 안정 버전을 우선 검토
- C# 기반 사용자 인터페이스는 WPF로 구현하며, 화면 계층은 MVVM 패턴을 적용하여 View, ViewModel, Model의 책임을 분리

## 4. 비기능 요구사항

- 가독성: 문서 구조를 일관되게 구성, 제목과 표를 사용하여 시각적으로 이해하기 쉽게 제작
- 유지보수성: 신규 조건 추가 시 확장 가능한 템플릿 구조 적용
- 정확성: Saramin과 JobKorea의 검색 조건 체계와 용어를 정확히 반영
- 접근성: 한국어 기반 사용자 중심 설명
- 확장성: Python 수집기와 C# 메인 서비스가 독립적으로 확장 가능해야 하며, 연동 포맷은 일관되게 유지
- 안정성: 수집 실패 시 메인 서비스 동작에 직접적인 영향을 주지 않도록 수집 계층과 제공 계층을 분리
- 기술 최신성: 프레임워크, 런타임, 주요 의존성은 구현 시점 기준 최신의 안정적인 버전을 사용하고, 실험적 버전은 기본 채택 대상에서 제외
- 구조 일관성: UI 계층은 MVVM 패턴을 기준으로 구현하여 화면 표시 로직과 비즈니스 로직의 결합도를 낮춤

## 5. 사용 사례

### 5.1 20대 취준생
- 조건: 신입, 수도권, 정규직, IT/개발 직무
- 기대 결과: 20대 연령층이 지원하기 좋은 기업 리스트, 주요 공고 요약

### 5.2 30대 경력직 구직자
- 조건: 경력직, 중견기업, 제조/생산 직무, 연봉 5천만 원 이상
- 기대 결과: 30대에 적합한 경력 채용 기업 정리 및 우선순위 정보

### 5.3 40대 이상 전환/재취업 희망자
- 조건: 경력직 또는 경력직 전환 가능, 지역 제한, 유연근무
- 기대 결과: 40대 이상 지원 가능한 기업과 채용 공고 특징 정리

### 5.4 50대 재취업 및 경력 활용 희망자
- 조건: 경력 중심, 안정적인 근무 형태, 지역 기반 채용, 관리/지원/전문직 우대
- 기대 결과: 50대 지원 가능 기업, 연령층 적합도, 주요 채용 조건 및 재취업 관점의 특징 정리

## 6. 데이터 소스 및 범위

- Saramin: 연령층, 경력, 근무 형태, 지역, 산업군 등 검색 필터
- JobKorea: 연령층, 경력, 근무 형태, 지역, 기업 규모 등 검색 필터
- Python 수집기 범위:
  - Saramin/JobKorea 검색 조건 수집
  - 플랫폼별 결과 데이터 정규화
  - 메인 서비스 전달용 중간 산출물 생성
- C# 메인 서비스 범위:
  - 수집 데이터 적재 및 조회
  - 연령층/조건 기반 추천 결과 제공
  - 문서화 및 API 응답 관리
- 범위:
  - 우선순위는 두 플랫폼의 공고 중 중복되거나 대표적인 기업
  - 연령층별 검색 결과에서 공통으로 유의미한 기업 우선 정리

## 7. 성공 기준

- DOCS 폴더 내에 연령층별 기업 정리 자료가 생성됨
- Saramin과 JobKorea 조건을 모두 반영한 비교 분석 가능
- 사용자 요구에 맞는 연령층별 추천 기업을 빠르게 확인할 수 있음
- 문서가 명확하고 유지보수 가능하도록 구성됨

## 8. 향후 확장 계획

- 자동화: Python 수집기를 통해 Saramin/JobKorea API 또는 웹 크롤링 기반 자동 데이터 수집 파이프라인 확장
- UI/UX: WPF 기반 사용자 인터페이스로 연령층 및 검색 조건 입력 후 결과를 시각화하며, 화면 구조는 MVVM 패턴을 적용
- 개인화: C# 메인 서비스에서 사용자 프로필 기반 추천 및 저장 기능 확장
- 다국어 지원: 영어 등 다국어 문서화 및 다국적 기업 채용 정보 확장

## 9. 초기 구현 범위

### 9.1 프로젝트 구조
- `src`: C# 메인 서비스 프로젝트 집합
- `tests`: C# 메인 서비스 테스트 프로젝트
- `tools`: Python 기반 수집기 및 보조 스크립트
- `DOCS`: PRD 및 운영/설계 문서

### 9.2 C# 메인 서비스 초기 구성
- `JobSearch.Api`: 사용자 요청을 처리하는 ASP.NET Core Web API
- `JobSearch.Desktop`: WPF 기반 메인 클라이언트
- `JobSearch.Application`: 연령층 기반 추천 유스케이스와 서비스 계층
- `JobSearch.Domain`: 핵심 엔티티, 값 객체, 인터페이스
- `JobSearch.Infrastructure`: 외부 데이터 적재, 파일 저장소, 향후 플랫폼 연동 구현
- `JobSearch.Contracts`: API 요청/응답 DTO

### 9.3 Python 수집기 초기 구성
- `tools/python-collector`: 플랫폼별 수집 로직 시작점
- 공통 설정, 수집 결과 모델, 정규화 로직, 파일 출력 기능을 포함
- 초기 단계에서는 메인 서비스 연동을 위해 JSON 파일 산출물을 생성

### 9.4 1차 구현 목표
- 메인 서비스 실행 및 기본 상태 확인 API 제공
- WPF 메인 화면에서 기본 검색 조건 입력과 추천 결과 표시 구조 준비
- 샘플 연령층 기반 추천 데이터 조회 API 제공
- Python 수집기에서 샘플 데이터 생성 및 JSON 저장 기능 제공
- 문서와 코드 구조가 이후 실제 플랫폼 연동을 수용할 수 있도록 준비

## 10. Phase 2 구현 상세

### 10.1 데이터 영속성 (SQLite)
- **도구:** SQLite, Entity Framework Core
- **목적:** 대량 수집 데이터의 효율적인 저장, 검색 성능 최적화, 수집기와 API 간의 비동기적 연동 자동화
- **DB 위치:** 프로젝트 루트의 `JobSearch.db` 파일 공유

### 10.2 실제 데이터 수집 (Python Crawler)
- **수집 대상:** 사람인(Saramin) 검색 결과 페이지
- **기술 스택:** `requests`, `BeautifulSoup4`
- **수집 항목:** 기업명, 공고 제목, 공고 상세 URL, 마감일, 연령대 정보 등
- **알고리즘:** 검색어 일치 및 연령대 가중치를 기반으로 한 적합도(Suitability Score) 계산

### 10.3 UI 및 편의성 고도화
- **하이퍼링크 연동:** 추천 리스트에서 공고 클릭 시 브라우저를 통해 원본 공고로 즉시 이동
- **상태 관리:** DB 연동 상태 및 마지막 수집 시간 표시
- **성능 최적화:** 대량 데이터 표시를 위한 DataGrid 가상화 및 비동기 로딩 강화
